# 技术交底书撰写指引

## 概述

本指引规定了技术交底书的撰写要求、格式标准和内容规范，旨在确保技术交底书的质量和合规性。

## 核心思维方法

### 三步法原则
技术交底书撰写遵循"三步法"底层思维：

| 步骤       | 内容     | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| **第一步** | 发现问题 | 识别现有技术的不足和缺点                 |
| **第二步** | 设计方案 | 通过技术手段解决现有技术存在的不足和缺点 |
| **第三步** | 技术效果 | 解决问题后取得的有益效果                 |

**核心公式：发现问题 + 设计方案 + 技术效果 = 专利技术方案**

## 发明人需提供的资料

### 必需资料
- **技术交底书**（包含实验方法、实验数据、对比例以及图片）

### 建议提供资料
- 三维图纸（stp格式）或 AutoCAD 图纸
- 图片或视频资料
- 相关的已申请专利文献（如本方案基于之前申请的方案）

## 文档结构要求

### 1. 基本结构
技术交底书应包含以下基本章节：

1. **技术领域** - 明确本专利所属的技术领域
2. **背景技术** - 描述现有技术的现状、存在的问题或不足（发现问题）
3. **发明摘要** - 概括本申请的基本方案和整体技术效果
4. **技术方案** - 详细描述本专利的技术方案，包括核心创新点（设计方案）
5. **创新关键点** - 说明本申请的保护重点和关键区别点
6. **有益效果** - 说明本专利带来的有益效果或优势（技术效果）
7. **变更设计方向** - 可能的替代或变形方案
8. **附图说明** - 描述附图内容（如有）
9. **具体实施方式** - 详细描述具体实施方式

### 2. 可选章节
根据需要可增加以下章节：
- 实施例描述
- 技术效果验证
- 应用场景说明
- 与其他技术的比较

## 各章节撰写要求

### 1. 技术领域
- 明确、具体地描述技术领域
- 避免过于宽泛或狭窄的描述
- 使用标准技术术语

### 2. 背景技术

#### 撰写指导
- **只写与本申请有关的技术背景以及现有技术、现有技术的缺点**
- 描述已有的与本申请相关的现有技术
- 说明现有技术的缺点，即"发现问题"
- 引用相关技术文献（如适用）
- 避免主观评价

#### 撰写要点
- 客观描述现有技术现状
- 明确指出现有技术存在的问题或不足
- 避免背景技术过于简单，确保能通过背景技术获知现有技术的缺点

#### 示例格式
```
目前[技术领域]大多采用[现有技术方案]。[现有技术方案]利用[技术原理]，
使[实现功能]。然而[现有技术方案]在使用过程中，存在[问题1]、[问题2]等问题。
```

### 3. 发明摘要

#### 撰写指导
- **不必描述实施细节**
- **概括说明本申请的方案或思路**
- **说明由这种方案能带来的技术效果**
- 针对现有技术存在的技术问题，提出本申请的基本方案或基本思路
- 说明本发明针对现有技术所具备的整体技术效果

#### 撰写要点
- 简要概括核心技术方案
- 说明解决的主要技术问题
- 阐述整体技术效果
- 与现有技术进行简要对比

#### 示例格式
```
本方案提供了一种[技术方案名称]，[简述核心设计]。这样一来，[技术效果]。
与现有技术相比，[优势说明]。
```

### 4. 技术方案

#### 撰写指导
- **应该结合机械图、流程图、原理框图、电路图、时序图进行说明**
- **充分说明本申请每一个详细技术方案**
- **本申请技术方案带来的效果的详细描述**：
  - 不仅要说明本申请整体能带来的效果
  - 由于某些技术特征可以直接导致的技术效果，也应该在这里结合技术特征的说明来描述效果
- **充分说明所有可以实现本申请的实施方式**：
  - 主要从实现本申请的细节角度考虑是否有相类似的方式方法或者其他变形的方法
  - 如果有，则请列出

#### 撰写要点
- 详细、完整地描述技术方案
- 突出核心创新点和技术特征
- 逻辑清晰，层次分明
- 结合附图进行说明
- 说明各技术特征的具体参数（如尺寸、形状、材料等）
- 说明各技术特征带来的具体技术效果

#### 附图要求
- 附图不限于形式，通过计算机绘制最好
- 手绘或照片也可以接受
- **重点是能表达出发明点**
- 附图应包含附图标记，与文字描述对应

### 5. 创新关键点

#### 撰写指导
- **本申请创新的关键点和想保护的技术方案，即保护重点**
- 与已知的现有技术相比，本申请会有一些与其不同的关键区别点
- 这些关键区别点通常会带来明显的或重要的技术效果
- **请说明哪些点是关键区别点，并将这些点可能的组合方案都列举出来**

#### 撰写要点
- 明确列出核心创新点
- 说明各创新点的技术效果
- 列举创新点的可能组合方案

### 6. 有益效果

#### 撰写要求
- 具体、客观地描述技术效果
- 可量化时应提供具体数据
- 与技术问题和解决方案相对应
- 避免夸大或虚假宣传

#### 撰写要点
- 整体技术效果
- 各技术特征带来的具体效果
- 与现有技术的对比优势

### 7. 变更设计方向

#### 撰写指导
- **站在竞争对手的角度可能提出的回避设计方案，即扩大保护范围**
- 从整体考虑，是否有整体的替代或变形方案
- 或者对核心技术特征的替代或变形方案
- 主要根据本发明体现出来的设计思路，别人能不能想到其他的整体的替代方案来实现

#### 撰写要点
- 列出可能的替代方案
- 说明核心技术特征的变形方式
- 考虑不同实施形式

### 8. 附图说明
- 清晰描述各附图的内容
- 附图编号与说明对应
- 说明附图与技术方案的关系
- **确保发明点有相关的附图以及附图标记**

### 9. 具体实施方式

#### 一般要求
- 提供具体、可行的实施示例
- 详细描述实施步骤和条件
- 可包括实验数据或测试结果
- 确保可重复性

#### 实验类方案特殊要求
- **实验技术方案清楚、完整**（如使用的生物材料、步骤、工艺参数、设备等）
- **对于必须依赖实验结果加以证实才能成立的发明专利申请**，需要在说明书中充分公开证明该技术方案成立的实验方法、实验数据等证据
- **对于制备一种产品的制备方法的发明**，在说明书中应提供证据来确认得到的目标产物，提供一定的实施例来佐证产品的功能和效果，必要时，提供一定的对比例以及图片

## 格式规范

### 1. 文档格式
- 文件格式：Markdown (.md) 或 Word (.docx)
- 编码：UTF-8
- 语言：中文

### 2. 字体和字号
- 正文：宋体，小四号（12pt）
- 标题：黑体，根据层级调整字号
- 英文和数字：Times New Roman

### 3. 段落和间距
- 行距：1.5倍行距
- 段落间距：段前0.5行，段后0.5行
- 首行缩进：2字符

### 4. 标题层级
- 一级标题：# 标题 （H1）
- 二级标题：## 标题 （H2）
- 三级标题：### 标题 （H3）
- 四级标题：#### 标题 （H4）

### 5. 编号和列表
- 使用有序列表（1., 2., 3.）表示步骤或顺序
- 使用无序列表（- 或 *）表示并列项
- 编号应连续、一致

### 6. 表格和图表
- 表格应有标题和编号
- 图表应清晰可读
- 图表应有相应的文字说明

## 术语和表达

### 1. 技术术语
- 使用标准、规范的技术术语
- 首次出现时应给出简要说明
- **全文术语使用应一致，同一个技术特征前文和后文命名应相同**

### 2. 语言表达
- 准确、简洁、明确
- 避免歧义和模糊表述
- 使用客观、中性的语言
- 避免使用营销性语言

### 3. 逻辑结构
- 各章节之间逻辑连贯
- 内容层次清晰
- 论点有充分的技术支持

## 常见问题及解决方案

### 1. 术语命名不一致
- **问题**：同一个技术特征，前文和后文命名不同，或者后文采用省略的命名
- **解决**：全文统一技术特征的命名，保持一致性

### 2. 发明点缺少附图
- **问题**：发明点没有相关的附图以及附图标记
- **解决**：为每个发明点提供对应的附图，并标注附图标记

### 3. 技术问题与效果缺失
- **问题**：只有技术方案，没有技术方案对应要解决的技术问题和取得的有益效果
- **解决**：遵循"三步法"，完整描述发现问题、设计方案、技术效果

### 4. 背景技术过于简单
- **问题**：背景技术过于简单，无法通过背景技术获知现有技术的缺点
- **解决**：详细描述现有技术的具体方案和存在的具体问题

### 5. 基于已有专利的申请
- **问题**：待申请的技术方案是基于之前申请过的方案进行申请
- **解决**：建议发明人提供之前申请过的专利文献作为参考

### 6. 实验方案不完整
- **问题**：实验技术方案描述不清楚、不完整
- **解决**：补充完整的实验方法、步骤、工艺参数、设备等信息

### 7. 缺乏实验证据
- **问题**：必须依赖实验结果加以证实的发明缺乏证据
- **解决**：在说明书中充分公开证明该技术方案成立的实验方法、实验数据等证据

### 8. 制备方法缺乏验证
- **问题**：制备方法类发明缺乏产物确认和效果验证
- **解决**：提供证据确认目标产物，提供实施例佐证产品功能和效果，必要时提供对比例及图片
